Shariah Compliance

Shariah Agent — Hard Gate Before Any AML Check

Verdict Condition Action
COMPLIANT Business activity and financing purpose clearly permissible under AAOIFI Set shariahCompliant: true — proceed to AML/KYB
REQUIRES_REVIEW Mixed business (5–25% prohibited revenue) or ambiguous sector Run AML/KYB, but force FlagComplianceForManualReview
NON_COMPLIANT Clearly prohibited sector (alcohol, riba, gambling) or invalid Murabahah structure Immediately exercise RejectCompliance — no API calls made

Technology Choice

Why Canton + Daml

What Canton Gives Us
  • Sub-transaction privacy — each party sees only their contracts, not each other's
  • Immutable audit trail baked into the ledger protocol
  • Bilateral contract execution — both borrower & institution must sign the Murabahah
  • Contract keys (LF 2.3) enforce one active onboarding per CAC number and one ApprovedProvider per FI party
  • Regulator observes only ApprovedProvider, MurabahahContract & PortfolioReport
What Daml Gives Us
  • Type-safe choice logic — invalid workflow transitions are compile-time errors
  • Role-based authorization enforced by the ledger, not application code
  • 20 templates across 7 modules covering the full lifecycle
  • Daml Script test suite — 226 tests across 5 test files, all passing
  • Import DAG: Types → FinancingProvider; Types → Compliance → Financing → Murabahah
"This isn't blockchain for the sake of it. Canton's privacy model is the only way to run a shared SME credit infrastructure where borrowers, institutions, and regulators coexist without leaking data to each other."
